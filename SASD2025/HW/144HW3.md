## 1) Utility Class  
### Class: System.Math  
Link: https://learn.microsoft.com/en-us/dotnet/api/system.math?view=net-10.0  

**วิเคราะห์:**  
System.Math จัดเป็น Utility Class เพราะเป็นคลาสที่รวมฟังก์ชันคณิตศาสตร์ไว้ให้เรียกใช้งาน เช่น Abs(), Round(), Sin(), Log() เป็นต้น  

เราไม่จำเป็นต้องสร้าง object ก่อนใช้งาน แต่สามารถเรียกใช้ได้เลย เช่น `Math.Round()`  

ลักษณะนี้แสดงให้เห็นว่าเป็นคลาสที่ทำหน้าที่เหมือนเครื่องมือสำเร็จรูปสำหรับช่วยคำนวณ ไม่ได้มีการเก็บสถานะข้อมูลของตัวเอง จึงจัดอยู่ในกลุ่ม Utility Class  

---

## 2) Mutable Class  
### Class: System.Text.StringBuilder  
Link: https://learn.microsoft.com/en-us/dotnet/api/system.text.stringbuilder?view=net-10.0  

**วิเคราะห์:**  
System.Text.StringBuilder จัดเป็น Mutable Class เพราะข้อมูลภายใน object สามารถเปลี่ยนแปลงได้หลังจากสร้างแล้ว  

เราสามารถใช้เมธอด เช่น Append(), Insert(), Remove() เพื่อแก้ไขข้อความเดิมได้เลย โดยที่ยังเป็น object เดิม ไม่ได้สร้าง object ใหม่  

ลักษณะนี้แสดงให้เห็นว่า object มีการเปลี่ยนแปลงสถานะภายในได้ จึงจัดเป็น Mutable Class  

---

## 3) Immutable Class  
### Class: System.String  
Link: https://learn.microsoft.com/en-us/dotnet/api/system.string?view=net-10.0  

**วิเคราะห์:**  
System.String จัดเป็น Immutable Class เพราะเมื่อสร้าง string แล้ว จะไม่สามารถเปลี่ยนแปลงค่าภายใน object เดิมได้  

ถ้ามีการแก้ไขข้อความ เช่น การต่อ string เพิ่ม ระบบจะสร้าง object ใหม่ขึ้นมาแทน ไม่ได้แก้ไขตัวเดิม  

ดังนั้นจึงถือว่า String เป็นคลาสที่ไม่สามารถเปลี่ยนแปลงค่าเดิมได้ หรือเรียกว่า Immutable  

---

## สรุป  

Utility Class ใช้สำหรับรวมฟังก์ชันหรือเครื่องมือช่วยทำงาน  
Mutable Class สามารถเปลี่ยนแปลงข้อมูลภายใน object ได้  
Immutable Class ไม่สามารถเปลี่ยนแปลงข้อมูลเดิมได้ ต้องสร้าง object ใหม่แทน  

ทั้งสามคลาสนี้เป็นตัวอย่างที่ชัดเจนของแนวคิด Utility, Mutable และ Immutable ใน .NET
